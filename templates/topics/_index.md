## v3 layout (current)

In v3.0, a TOPIC is a FOLDER, not a single file:

```
workspaces/<ws>/<slug>/
├── 00-README.md          ← MOC (TL;DR + Aspects auto-list + Cross-links manual)
├── YYYY-MM-DD-<aspect>.md ← dated aspect files (multiple per topic)
└── lessons.md            ← per-topic ledger (5-field schema)
```

Reserved subdirs at workspace root (cannot be used as topic slugs):
`docs`, `journal`, `skills`, `research`.

Reserved aspect names (cannot be used as `<aspect>`):
`readme`, `lessons`, `00-readme`.

Slug rules (BOTH topic slug and aspect slug):
- Regex `^[a-z0-9][a-z0-9-]{0,59}$`
- Kebab-case, ≤60 chars
- Unique within workspace
- Never rename a published slug (breaks wikilinks)

## How topics are created

- `mems` / `/mem-save` / `memT` route entries through `_topic.py` which:
  1. Picks the topic folder via keyword overlap on `00-README.md` headings.
  2. Creates `workspaces/<ws>/<slug>/00-README.md` if none matches (≥3 keywords).
  3. Appends today's entry to `workspaces/<ws>/<slug>/YYYY-MM-DD-<aspect>.md`.
- `[secret-ref]` lines side-channel to `shared/secrets.md`.
- `[skill-ref]` lines side-channel to `workspaces/<ws>/skills/<slug>.md`.

## Wikilink resolution

`[[slug]]` resolves in this order:
1. v3 landing: `<base>/<slug>/00-README.md`
2. rglob v3 `*/00-README.md` where `slug:` matches
3. v2.4 legacy: `<base>/<slug>/<slug>.md` (folder-note)
4. rglob v2.4 `*/<slug>.md` folder-note
5. v2.3 legacy: `<base>/<slug>.md` flat
6. rglob v2.3 `<slug>.md`

Cross-workspace: `[[ws:slug]]`. Shared: `[[shared:secrets]]`.
